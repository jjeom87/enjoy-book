## Chapter 03 라우터로 내비게이션 구현하기
### 3.1 라우터
#### 3.1.1 위치 정책(Location strategy)
#### 3.1.2 라우터 구성 요소
* Router
  * 라우터를 구현하는 객체
  * 원하는 경로로 이동하려면 이 객체의 navigate() 함수와 navigateByUrl() 함수를 사용하거나, 라우터 객체의 path 프로퍼티 값을 변경
  * 브라우저의 URL을 직접 수정해도 원하는 경로로 이동할 수 있음
* RouterOutlet
  * 라우터가 컴포넌트를 렌더링하는 영역인 <router-outlet>을 구현하는 디렉티브
* Routes
  * 특정 URL에 연결되는 컴포넌트를 지정하는 배열
* RouterLink
  * HTML 앵커 태그(<a>)의 원래 용도는 브라우저의 URL 주소를 변경하는 것
  * RouterLink 디렉티브를 사용하면 앵커 태크가 Angular 라우터를 통하도록 기능을 변경 할 수 있음
  * RouterLink 사용하면 라우터가 렌더링할 컴포넌트에 인자를 전달할 수 있음
* ActivateRoute
  * 현재 동작하는 라우터 인스턴스를 가리키는 객체

## Chapter 04 의존성 주입
### 4.1 의존성 주입과 제어권 역전
#### 4.1.1 의존성 주입 패턴
* 객체 A가 객체 B에 의존성이 있다고 할 떄, 객체 A는 객체 B의 인스턴스를 직접 생성하는 대신, "나는 B 형식의 객체가 필요한데 누군가 만들어 주지 않을래?" 라고 요청
#### 4.1.2 제어권 역전 패턴
* 애플리케이션이 프레임워크에 있는 API를 직접 사용하는 대신, 프레임워크가 객체를 만들고 애플리케이션에 전달하는 방식
#### 4.1.3 의존성 주입의 장점
* 의존성 주입 패턴을 사용하면 코드의 결합도를 낮출 수 있고, 원하는 코드만 때어내서 테스트할 수 있으며, 재사용하기 쉬운 코드를 작성할 수 있음

#### 5.2.1 옵저버와 옵저버블이 무엇인가?
* 옵저버블은 연속된 데이터 스트림을 생성하는 객체
  * 콜드 옵저버블 구독자가 있는 경우에만 스트리밍 데이터를 만들기 시작
  * 핫 옵저버블은 데이터를 받는 구독자가 없더라도 데이터를 스트리밍
* 옵저버는 이 스트림을 구독해서 사용하는 객체
* 옵저버블 스트림
  * 다음 엘리먼트를 전달한다.
  * 어레를 전달한다.
  * 스트리밍이 종료되었다는 신호를 전달한다.
* 옵저버
  * 다음 엘리먼트를 받았을 때 처리하는 함수
  * 에러를 받았을 때 처리하는 함수
  * 데이터 스트림이 끝났을 때 실행하는 함수
